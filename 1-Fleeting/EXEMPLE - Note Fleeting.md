# EXEMPLE - Idée rapide sur les commits atomiques

> [!info] Note rapide
> Cette note doit être traitée et transformée en note permanente ou archivée.

## Contenu

Idée : Les commits Git devraient être atomiques comme les notes Zettelkasten. Un commit = une fonctionnalité/correction logique.

Pourquoi ? Facilite le revert, le cherry-pick, la revue de code.

Exemple : Ne pas mélanger "fix bug + add feature" dans un seul commit.
